# 아스트로펫 티저 페이지(10/step-1) 코드 리뷰

이 문서는 `10/step-1` 브랜치(PR #1)에서 생성된 티저 페이지 구현체에 대한 리뷰 및 피드백입니다. 현재 `main` 브랜치에 병합하기 전, 다음 사항들을 반드시 개선해야 합니다.

## 🔴 주요 크리티컬 이슈 (개선 필수)

1. **프로젝트 구조 덮어쓰기 오류**
   - **문제**: 이미 구축되어 있던 `v0.3.0` 환경을 인지하지 못하고, 빈 디렉토리로 간주하여 `create-next-app`으로 Next.js 프로젝트를 통째로 다시 초기화했습니다. 이로 인해 기존 `main` 코드와 심각한 병합 충돌(Merge Conflict)이 발생합니다.
   - **해결 방안**: PR을 닫고, 현재 완전한 코드가 있는 최신 `main` 브랜치를 기준으로 분기(`11/review` 등)하여 오직 `src/app/teaser/page.tsx`와 필요한 컴포넌트(예: `EggSvg.tsx`)만 기존 프로젝트 위에 추가하는 방식으로 재작업해야 합니다.

2. **React Hydration Mismatch 위험 (`Stars.tsx`)**
   - **문제**: 새롭게 작성된 `Stars.tsx`에서 렌더링 시 `Math.random()`을 직접 사용했습니다. 이는 서버에서 생성된 HTML과 클라이언트 렌더링 결과가 달라져 Next.js에서 Hydration Error를 유발합니다.
   - **해결 방안**: 기존 `main` 브랜치에 이미 존재하며 의사 난수(Pseudo-random)를 통해 결정적으로 동작하는 안전한 `Stars.tsx` 컴포넌트를 그대로 재사용해야 합니다.

3. **CSS 전역 오버라이드 및 중복 구현**
   - **문제**: `globals.css`를 통째로 다시 작성하여, 기존 게임에 정의되어 있던 중요 유틸리티 클래스와 키프레임들이 삭제될 위험이 있습니다.
   - **해결 방안**: 기존 `globals.css`의 토큰(예: `--color-space-dark` 등)을 그대로 활용하고, 둥둥 뜨는 효과(`anim-bob`)가 필요하다면 기존 애니메이션을 활용하거나 기존 파일에 추가로 덧붙이는 형태로 구현해야 합니다.

## 🟢 긍정적인 점 및 유지할 부분

- **디자인 방향성 및 모바일 퍼스트**: `max-w-[430px]` 제약과 중앙 정렬을 활용한 레이아웃 구성이 모바일 PWA를 지향하는 아스트로펫의 컨셉과 매우 잘 맞습니다.
- **감성적인 카피라이팅 및 UI**: 다크 우주 배경과 핑크/민트 포인트를 활용해 CTA 폼과 안내 문구를 구현한 부분은 사용자에게 따뜻한 힐링 감성을 전달하기에 훌륭합니다.
- **SVG 활용**: 외부 이미지 에셋 없이 SVG와 CSS 그라디언트만으로 `EggSvg`를 귀엽게 그려낸 점은 프로젝트 가이드라인(Flat/SVG 스타일)을 매우 잘 준수했습니다.
